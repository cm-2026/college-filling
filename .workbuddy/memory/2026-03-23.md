#### 2. 显示逻辑

  - 有推荐结果时显示"更多推荐"按钮
  - 无推荐结果时隐藏按钮
  - 通过 `style.display = 'inline-block'` 控制显示

#### 3. 加载更多功能
**实现逻辑**:
1. 扩大分数范围(当前分数+10分)
2. 从数据库查询更大范围的院校
3. 过滤掉已显示的推荐(避免重复)
4. 将新推荐追加到现有结果
5. 重新显示所有推荐

**核心代码**:
```javascript
document.getElementById('moreRecommendationsBtn').addEventListener('click', async function() {
    // 扩大分数范围查询更多推荐
    const formData = getFormData();
    const expandedScore = formData.score + 10; // 扩大10分范围

    // 从数据库查询
    const response = await fetch(`${api.baseURL}/recommend-from-db`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            score: expandedScore,
            subjectCombination: formData.subjectCombination,
            batch: formData.batch,
            region: formData.region
        })
    });

    const result = await response.json();

    // 过滤掉已经显示的结果
    const existingScores = new Set(currentRecommendation.recommendations.map(r => r.score));
    const newRecommendations = result.data.filter(r => !existingScores.has(r.score));

    // 更新推荐结果
    currentRecommendation.recommendations = [...currentRecommendation.recommendations, ...newRecommendations];

    // 重新显示所有推荐
    displayResults(currentRecommendation.recommendations);
});
```

#### 4. 用户体验优化

**状态提示**:
- 点击时显示 "正在加载更多推荐..."
- 成功后显示 "已加载 X 条更多推荐"
- 无新推荐时显示 "没有更多推荐的院校了"

**去重处理**:
- 使用 Set 存储已显示院校的分数
- 通过分数过滤避免重复推荐
- 确保每次点击都加载新的推荐

**无缝追加**:
- 新推荐追加到现有结果列表
- 保持原有的排序和分类
- 重新渲染所有推荐,保证一致性

### 功能特点

1. **渐进式加载**: 用户可以逐步加载更多推荐,不会一次性显示过多信息
2. **智能去重**: 自动过滤已显示的院校,避免重复
3. **灵活范围**: 每次扩大10分范围,逐步扩展推荐范围
4. **实时反馈**: 加载状态和结果的即时提示
5. **无缝体验**: 新推荐追加到现有结果,保持上下文连续性

## 选科按钮点击和地区切换功能修复（2026-03-23 18:31）

**时间**: 2026-03-23 18:31

**用户反馈**: 
- 选科按钮没办法点击
- 选科组合没办法根据地区进行切换

### 问题分析

#### 问题 1：事件监听器重复添加

**现象**：
- 地区改变时，虽然选科界面重新渲染，但选科标签无法点击
- 或者点击后没有反应

**根本原因**：
在 `setupRegionChangeListener()` 函数中，地区 select 元素的 change 事件监听器可能被添加多次：

```javascript
function setupRegionChangeListener() {
    const regionSelect = document.getElementById('region');
    regionSelect.addEventListener('change', function() {
        // 重新渲染选科界面
        renderSubjectSelection();
        // ...
    });
}
```

如果此函数被多次调用，就会有多个相同的事件监听器监听同一个元素。虽然代码中 `initSubjectTags()` 只调用一次，但某些路径（如加载历史记录）可能导致监听器重复。

#### 问题 2：监听器冲突

当多个监听器都被触发时，可能会导致：
- 事件处理代码重复执行
- 状态更新冲突
- DOM 操作竞争

### 修复方案

**文件**：`e:\xm\index-mysql.html`  
**行数**：1163-1181

**修复策略**：使用 `cloneNode()` 替换节点，移除旧的所有事件监听器

**修复前**：
```javascript
function setupRegionChangeListener() {
    const regionSelect = document.getElementById('region');
    regionSelect.addEventListener('change', function() {
        // ...
    });
}
```

**修复后**：
```javascript
function setupRegionChangeListener() {
    const regionSelect = document.getElementById('region');
    
    // 移除之前的监听器（如果有的话），防止重复添加
    const newRegionSelect = regionSelect.cloneNode(true);
    regionSelect.parentNode.replaceChild(newRegionSelect, regionSelect);
    
    // 给新的select添加监听器
    document.getElementById('region').addEventListener('change', function() {
        // 清空选科选择
        document.getElementById('subjectCombination').value = '';
        document.getElementById('resultsSection').classList.remove('show');

        // 重新渲染选科选择界面（这会自动调用 setupSubjectTagEvents）
        renderSubjectSelection();
        
        // 重新清除之前的错误提示
        document.getElementById('subjectError').style.display = 'none';

        // 显示提示
        showToast('选科模式已根据地区自动切换', 'info');
    });
}
```

**技术说明**：
1. `cloneNode(true)` 创建元素的完整副本，包括所有属性
2. 新克隆的元素不包含任何事件监听器
3. 用新元素替换旧元素，这样就移除了旧元素上的所有监听器
4. 给新元素添加单个新的监听器

### 修复原理

**事件流程（修复后）**：

```
页面加载时：initSubjectTags()
├─ renderSubjectSelection()          # 渲染选科标签
│  └─ setupSubjectTagEvents(mode)    # 绑定标签点击事件 ✅
└─ setupRegionChangeListener()       # 添加地区change事件
   └─ [克隆select移除旧监听器]      # 确保只有一个监听器

用户改变地区选择：
├─ region change 事件触发（只有1个监听器）
├─ 清空选科选择
├─ renderSubjectSelection()          # 重新渲染选科标签
│  └─ setupSubjectTagEvents(mode)    # 重新绑定标签点击事件 ✅
└─ 显示提示

用户点击选科标签：
├─ tag click 事件触发
├─ 更新选中状态（添加/移除selected类）
├─ 更新隐藏字段值（subjectCombination）
└─ 验证选择是否完整
```
