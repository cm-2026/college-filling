## 修复选科按钮初始化失败问题（09:20）

**用户反馈**：选科按钮无法点击，页面右下角显示"初始化失败"。

**根本原因**：
`mode312`（3+1+2 新高考模式，即河南默认模式）的 HTML 模板中**缺少** `<input id="subjectCombination">` 和 `<div id="subjectError">` 两个元素。

`renderSubjectSelection()` 调用 `container.innerHTML = html` 覆盖了容器内容（包括初始 HTML 里的这两个元素），而 mode33 和 modeTraditional 模板都有重新生成这两个元素，唯独 mode312 模板没有。导致 `setupSubjectTagEvents()` 中 `errorDiv.textContent = ...` 报错（errorDiv 为 null），触发 DOMContentLoaded 的 catch，弹出"系统初始化失败"toast，整个选科模块无法工作。

**修复内容**（e:/xm/index-mysql.html）：
1. 在 `mode312` 的 HTML 模板开头补充 `<input type="hidden" id="subjectCombination" required>` 和 `<div class="invalid-feedback" id="subjectError">` 两个元素。
2. 在 `setupSubjectTagEvents` 函数开头添加空值防御检查，避免未来再因此类问题崩溃。

**状态**：✅ 已修复

## 修复目标地区偏好字符集错误（10:14）

**用户反馈**：输入"北京"报错：`Conversion from collation utf8mb4_unicode_ci into latin1_swedish_ci impossible for parameter`

**问题原因**：后端 MySQL 连接池配置未指定字符集，默认使用 `latin1_swedish_ci`，而数据库表使用 `utf8mb4_unicode_ci`，导致字符集冲突。

**修复内容**（e:/xm/backend/server.js）：
- 第27行：在 `createPool` 配置中添加 `charset: 'utf8mb4'`

**技术细节**：
- MySQL 连接默认使用 `latin1` 字符集
- 数据库 gaokao 和表 admission 使用 `utf8mb4` 字符集
- 查询时需要将中文参数（如"北京"）从 utf8mb4 转换到 latin1，导致字符集转换错误

**状态**：✅ 已修复并重启后端
