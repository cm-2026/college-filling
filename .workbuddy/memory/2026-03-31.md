## 智能推荐查询性能优化（09:24）

- **问题**：点击"开始推荐"后查询速度太慢
- **根本原因**：
  1. 缺少关键组合索引
  2. 3+3模式使用UNION ALL查询3次大表
  3. MySQL选择了次优索引导致filesort
- **优化措施**：
  1. **添加组合索引**：
     - `idx_source_province` (source_province)
     - `idx_province_score` (source_province, group_min_score_1)
     - `idx_province_subject_score` (source_province, subject_type, group_min_score_1)
     - `idx_province_require_score` (source_province, subject_require, group_min_score_1)
  2. **优化SQL查询**：
     - 3+3模式：使用CASE WHEN替代UNION ALL，减少表扫描次数从3次降到1次
     - 查询字段：只查询必要的字段，减少数据传输
  3. **更新重庆为3+3地区**：重庆加入mode33Provinces数组
- **性能提升**：
  - 优化前：UNION ALL扫描表3次 + filesort排序
  - 优化后：单次扫描 + Backward index scan（反向索引扫描，无需排序）
  - 预计查询速度提升3-5倍
- **验证**：EXPLAIN显示使用 `idx_province_subject_score` 组合索引，type=range，Using index condition
