function hasThumb(majors, schoolFeatures) {

## 专业组查询数量不一致问题修复（2026-04-10 10:39）

### 问题现象

用户反馈：`西安建筑科技大学` 专业组102，SQL直接查询是21条，但页面显示25条。

### 问题原因

用户没有完整选择选科组合（只选了物理，没选再选科目），导致：
1. 前端 `updateCombo()` 函数判断选科不完整，设置 `subjectCombination = ''`
2. 后端收到空字符串，不应用 `subject_type` 筛选
3. 返回该专业组的所有25条数据（包括4条历史类 + 21条物理类）

### 解决方案

**前端修改**（`e:\xm\js\index-mysql.js` showGroupModal函数）：
- 即使 `subjectCombination` 为空，也尝试从已选标签中提取必选科目（物理/历史）
- 只要有必选科目，就用来筛选
- 添加调试日志提示用户

**后端修改**（`e:\xm\backend\server.js` /api/school-group-majors）：
- 将 `mode33Provinces` 变量定义移到前面（第1189行），避免引用错误
- 添加详细的调试日志（SQL、参数、结果统计）

### 测试验证

| 测试场景 | 结果 | subject_type分布 |
|---------|------|-----------------|
| 完整选科（物理,化学,生物） | ✅ 21条 | 物理:21 |
| 只有必选（物理） | ✅ 21条 | 物理:21 |
| 空字符串 | ✅ 25条 | 历史:4, 物理:21 |
| 历史类选科 | ✅ 4条 | 历史:4 |

### 技术要点

- 3+1+2模式选科逻辑：必选1门（物理/历史）+ 再选2门
- 只有完整选择3门，`updateCombo()` 才会设置 `subjectCombination` 值
- 前端修改后，即使不完整，只要有必选科目也能正确筛选

### 3. JS 悬浮逻辑

```javascript
// 院校描述 Tooltip
const schoolTooltip=document.getElementById('schoolTooltip');
const schoolTtTitle=document.getElementById('schoolTtTitle');
const schoolTtBody=document.getElementById('schoolTtBody');
const schoolCache={};
let schoolHideTimer=null, pendingSchool='';

function showTooltipSchool(e, schoolName){
    clearTimeout(schoolHideTimer);pendingSchool=schoolName;
    schoolTtTitle.textContent=schoolName;
    positionSchoolTooltip();
    schoolTooltip.classList.add('visible');
    if(schoolCache[schoolName]!==undefined){renderSchoolTooltipContent(schoolCache[schoolName]);return;}
    schoolTtBody.innerHTML='<div class="school-tooltip-loading">加载中…</div>';
    fetch(`${API_BASE}/school-detail?schoolName=${encodeURIComponent(schoolName)}`)
        .then(r=>r.json())
        .then(res=>{
            if(res.success&&res.data)schoolCache[schoolName]=res.data;
            if(pendingSchool===schoolName){renderSchoolTooltipContent(res.success?res.data:null);schoolTooltip.classList.add('visible');}
        })
        .catch(err=>{if(pendingSchool===schoolName)schoolTtBody.innerHTML=`<div class="school-tooltip-none">加载失败</div>`;});
}
```

function truncateDescription(text, maxLen=500){
    if(!text) return '';
    let result = text.substring(0, maxLen);

    if(result.length < text.length && !result.endsWith('。')){

        if(lastPeriod > -1 && lastPeriod < maxLen + 100){
            result = text.substring(0, lastPeriod + 1);
        }
    }
    return result;
}

- P = 99 × e^(-位次差 / k)
- R：用户位次
- M：专业最低位次
- 约束：结果永远 < 99%

### 修改内容

**修改文件**: `e:\xm\backend\server.js`

**核心公式**:
```javascript
位次差 = 专业最低位次 - 用户位次
k = 1831  // 校准参数
P = 99 × e^(-位次差 / k)
```

**效果展示**:
- 位次差 = 0（压线）：≈99%
- 位次差 = 1500（领先1500位）：≈60%
- 位次差 = 3000（领先3000位）：≈36%

**推荐类型判断**:
- 保：领先3000位以上（概率36%以下，极稳妥）
- 稳：压线到领先3000位（概率36%-99%）
- 冲：位次落后（录取概率低）

**降级方案**:
无位次数据时使用分数差计算（旧逻辑）

**优势**:
1. 更符合实际录取规律（位次比分数更准确）
2. 指数衰减模型，避免线性模型的突兀
3. 压线时概率最高（99%），随位次差距增大递减
4. 保留分数差降级方案，兼容性好
