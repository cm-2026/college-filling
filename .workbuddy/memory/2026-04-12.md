## 修复Excel导出文件无法打开问题（2026-04-12 09:40）

**问题描述**：导出的Excel文件无法打开，提示"文件格式或文件扩展名无效"

**根本原因**：
1. Python脚本`export_excel.py`中使用了emoji字符（✅❌📖📊）
2. Windows系统默认使用GBK编码
3. emoji字符无法在GBK编码下输出，导致`print()`抛出`UnicodeEncodeError`
4. 脚本执行失败，Excel文件生成不完整

**修复方案**：
1. 将所有emoji字符替换为纯ASCII文本（如`[OK]`、`[ERROR]`、`[INFO]`）
2. 将`exec()`改为`execSync()`同步执行，确保文件完整生成后再发送
3. 添加文件大小验证，防止发送损坏的文件
4. 使用时间戳生成唯一文件名，避免并发冲突

**修改文件**：
- `backend/export_excel.py`：移除emoji，添加wb.close()
- `backend/server.js`：改用execSync同步执行，添加详细日志和错误处理

**验证**：测试数据成功生成5486字节的Excel文件，Python openpyxl可正常读取

## 修复甘肃查询无数据问题（2026-04-12 13:40）

**问题描述**：甘肃501分物理化学生物查询无数据

**根本原因**：
1. 后端使用位次范围查询（501分位次28267 ~ 491分位次32386）
2. 查询返回位次在范围内的记录（如min_rank_1=28609）
3. 但该记录的min_score_1=511，高于用户分数501
4. 前端过滤逻辑 `minScore <= userScore` 把它过滤掉了

**数据不一致**：
- score_rank_table：511分对应位次24342
- admission_plan：该专业min_score_1=511，min_rank_1=28609
- 原因：数据来源不同年份或批次

**修复方案**：
- 修改前端过滤逻辑，使用 `group_min_score_1`（专业组最低分）
- 允许分数高出10分以内（"冲"的范围）
- 过滤条件：`minScore <= userScore + 10`

**修改文件**：`js/index-mysql.js` 第437-442行

**最终方案**：完全移除前端分数过滤，直接显示后端位次范围查询的所有结果
