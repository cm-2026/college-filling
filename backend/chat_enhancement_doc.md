# 智能客服增强功能说明

## 功能概述

智能客服现在可以根据用户的提问类型,智能调用系统现有的查询功能,返回数据库中的真实数据,而不是完全依赖AI生成。

## 工作原理

1. **用户提问** → 系统分析意图
2. **意图匹配** → 调用相应的查询接口
3. **数据库查询** → 返回真实数据
4. **格式化输出** → 生成友好的回复

## 支持的查询类型

### 1. 院校推荐
- **触发条件**: 用户询问"推荐院校"、"有什么院校推荐"等
- **查询接口**: `/api/recommend-from-db`
- **参数**: 省份、分数、选科组合
- **返回**: 院校推荐列表(前5-10所)

**示例**:
```
用户: 推荐院校
系统: 📊 河南省 500分 历史政治地理组合推荐院校：
      1. 🏫 防灾科技学院
      2. 🏫 安阳师范学院
      ...
```

### 2. 专业推荐
- **触发条件**: 用户询问"推荐专业"、"有什么专业推荐"等
- **查询接口**: 新增的 `queryMajorRecommendations` 函数
- **参数**: 选科组合
- **返回**: 专业推荐列表(按专业类分组,每类3个)

**示例**:
```
用户: 推荐专业
系统: 📚 根据你的选科 历史政治地理，推荐以下专业方向：

      1. **历史学类**
         1. 历史学 (暂无, 0所院校)
         2. 世界历史 (暂无, 0所院校)
         3. 考古学 (暂无, 0所院校)

      2. **政治学类**
         1. 思想政治教育 (暂无, 0所院校)
         ...
```

### 3. 院校详情
- **触发条件**: 用户询问"XX院校介绍"、"XX大学怎么样"等
- **查询接口**: `/api/major-detail` (通过学校名称搜索)
- **参数**: 院校名称
- **返回**: 院校基本信息

### 4. 专业详情
- **触发条件**: 用户询问"XX专业介绍"、"XX专业怎么样"等
- **查询接口**: `/api/major-detail`
- **参数**: 专业名称
- **返回**: 专业介绍、就业前景、开设院校

### 5. 一般问题
- **触发条件**: 其他问题
- **处理方式**: 调用 DeepSeek AI 生成回复
- **示例**: "怎么填志愿"、"冲稳保策略"

## 实现细节

### 新增函数

#### `queryMajorRecommendations(subjectCombination)`
根据选科组合推荐专业

**参数**:
- `subjectCombination`: 选科组合字符串,如"历史,政治,地理"

**返回**:
```javascript
[
  {
    category: "历史学类",
    majors: [
      { major_name: "历史学", min_score_1: 550, school_count: 120 },
      { major_name: "世界历史", min_score_1: 530, school_count: 85 },
      { major_name: "考古学", min_score_1: 500, school_count: 45 }
    ]
  },
  // ... 更多专业类
]
```

### 修改的函数

#### `tryEnhancedDbQuery(text, context, userId)`
- 在 `case 'recommend'` 中添加专业推荐逻辑
- 当用户询问推荐专业时,调用 `queryMajorRecommendations` 函数

#### `formatEnhancedResponse(intentType, enhancedData, params)`
- 在 `case 'recommend'` 中添加专业推荐的格式化逻辑
- 按专业类分组展示,每类最多3个专业

## 优势

1. **数据准确**: 使用系统数据库中的真实数据
2. **响应快速**: 数据库查询比AI生成更快
3. **成本低**: 减少AI API调用次数
4. **可维护**: 查询逻辑清晰,易于优化

## 未来优化方向

1. 根据分数段推荐专业(如500-510分推荐哪些专业)
2. 根据就业前景推荐专业(如哪些专业就业率高)
3. 根据院校特色推荐专业(如XX大学以XX专业闻名)
4. 添加专业就业率、薪资水平等数据
