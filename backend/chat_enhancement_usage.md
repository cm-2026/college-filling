## 技术实现

### 新增函数

**`queryMajorRecommendations(subjectCombination)`**
- 根据选科组合查询匹配的专业
- 按专业类分组
- 返回每类前3个专业

**文件位置**: `backend/server.js` 第 3216-3265 行

### 修改的函数

**`tryEnhancedDbQuery(text, context, userId)`**
- 在 `case 'recommend'` 中添加专业推荐逻辑
- 当用户询问推荐专业时,调用 `queryMajorRecommendations`

**文件位置**: `backend/server.js` 第 3804-3811 行

**`formatEnhancedResponse(intentType, enhancedData, params)`**
- 添加专业推荐的格式化逻辑
- 按专业类分组展示

**文件位置**: `backend/server.js` 第 4069-4110 行

## 数据查询逻辑

专业推荐查询使用以下逻辑:

1. **三科组合** (3+3模式):
   - 匹配选科组合,如"历史和地理和政治"

2. **两科组合** (3+1+2模式):
   - 匹配精确选科组合,如"历史和政治"

3. **单科组合**:
   - 匹配单科要求,如"历史"

4. **排序规则**:
   - 按开设院校数量降序
   - 专业名称升序

5. **返回限制**:
   - 最多15个专业
   - 按专业类分组,每类最多3个

## 数据来源

- `major_info`: 专业基本信息
- `admission_plan`: 录取计划(用于获取录取分数和开设院校数)

## 故障排查

### 问题1: 没有找到匹配的专业

**原因**: 数据库中该选科组合的专业数据较少

**解决方案**:
1. 检查 `major_info` 表是否有数据
2. 检查 `admission_plan` 表是否有录取数据
3. 尝试放宽选科要求

### 问题2: 查询返回空结果

**原因**:
1. 选科组合格式不正确
2. 数据库连接问题

**解决方案**:
1. 检查选科组合格式,如"历史,政治,地理"
2. 检查数据库连接配置

### 问题3: 服务器启动失败

**原因**: 端口3000已被占用

**解决方案**:
```bash
# Windows: 查找占用端口的进程
netstat -ano | findstr :3000

# 杀死进程
taskkill /PID <进程ID> /F
```
