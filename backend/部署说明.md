## 🚀 部署步骤

### 第1步：安装Node.js依赖

由于npm缓存权限问题，请按以下方式手动安装：

#### 方法A：使用管理员权限安装（推荐）

1. 右键点击"开始菜单"，选择"Windows PowerShell (管理员)"或"命令提示符(管理员)"
2. 执行以下命令：

```bash
cd e:\xm\backend
npm install express mysql2 cors --no-optional
```

#### 方法B：更改npm缓存目录

```bash
# 创建新的缓存目录
mkdir e:\npm-cache

# 设置npm缓存目录
npm config set cache e:\npm-cache --global

# 重新安装依赖
cd e:\xm\backend
npm install
```

#### 方法C：使用Yarn安装（如果已安装Yarn）

```bash
cd e:\xm\backend
yarn install
```

### 第2步：配置MySQL数据库

#### 2.1 确保MySQL服务已启动

1. 打开"服务"管理器（services.msc）
2. 找到"MySQL"或"MySQL80"服务
3. 确保状态为"正在运行"

如果未运行，右键点击"启动"。

#### 2.2 创建数据库

使用MySQL客户端（如Navicat、MySQL Workbench或命令行）执行以下SQL：

```sql
-- 方式1：使用database.sql文件
cd e:\xm\backend
mysql -u root -pcm1990131 < database.sql

-- 方式2：手动执行SQL
CREATE DATABASE IF NOT EXISTS college_recommendation CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE college_recommendation;

CREATE TABLE IF NOT EXISTS recommendations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    region VARCHAR(50) NOT NULL COMMENT '地区',
    score INT NOT NULL COMMENT '高考分数',
    subjectCombination VARCHAR(50) NOT NULL COMMENT '选科组合',
    batch VARCHAR(20) NOT NULL COMMENT '报考批次',
    targetRegion VARCHAR(100) DEFAULT '' COMMENT '目标地区偏好',
    majorPreference VARCHAR(50) DEFAULT '' COMMENT '专业方向意向',
    graduationPlan VARCHAR(20) DEFAULT '' COMMENT '毕业意向规划',
    personality VARCHAR(20) DEFAULT '' COMMENT '性格类型',
    otherRequirements TEXT COMMENT '其他要求',
    recommendations JSON COMMENT '推荐结果（JSON格式）',
    generateTime DATETIME NOT NULL COMMENT '生成时间',
    saveTime TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '保存时间',
    
    INDEX idx_score (score),
    INDEX idx_region (region),
    INDEX idx_saveTime (saveTime)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='志愿推荐记录表';
```

**数据库连接信息已配置在server.js中：**
- 主机：localhost
- 用户名：root
- 密码：cm1990131
- 数据库：college_recommendation

## 🔧 常见问题解决

### 问题1：npm install 权限错误

**错误信息：**
```
npm error code EPERM
npm error syscall open
npm error errno EPERM
```

**解决方案：**
1. 以管理员身份运行命令行
2. 更改npm缓存目录（见方法B）
3. 删除node_modules和package-lock.json后重试

```bash
cd e:\xm\backend
rmdir /s /q node_modules  # Windows命令
del package-lock.json
npm install
```

### 问题2：MySQL连接失败

**错误信息：**
```
❌ 数据库连接失败: Access denied for user 'root'@'localhost'
```

**解决方案：**
1. 检查MySQL用户名和密码是否正确
2. 确保MySQL服务已启动
3. 检查防火墙设置，确保端口3306开放
4. 在server.js中修改数据库配置：

```javascript
const dbConfig = {
  host: 'localhost',
  user: 'root',
  password: 'cm1990131',  // 修改为你的密码
  database: 'college_recommendation'
};
```

### 问题3：端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案：**
修改server.js中的端口号：

```javascript
const PORT = 3001;  // 改为其他未使用的端口
```

### 问题4：CORS跨域错误

**错误信息：**
```
Access to fetch blocked by CORS policy
```

**解决方案：**
1. 确保后端已启动CORS中间件（server.js中已配置）
2. 检查前端API地址是否正确
3. 确保前后端端口一致

### 问题5：中文乱码

**解决方案：**
1. 确保MySQL数据库字符集为utf8mb4
2. 确保HTML文件包含charset声明
3. 确保server.js中设置了正确的字符编码

## 🔒 安全建议

### 生产环境部署

1. **修改数据库密码**：使用更复杂的密码
2. **创建专用数据库用户**：不要直接使用root
3. **配置环境变量**：不要在代码中硬编码密码
4. **使用HTTPS**：生产环境建议使用HTTPS
5. **限制访问IP**：配置MySQL允许访问的IP地址

### 示例：使用环境变量配置

```javascript
// 修改server.js中的数据库配置
const dbConfig = {
  host: process.env.DB_HOST || 'localhost',
  user: process.env.DB_USER || 'root',
  password: process.env.DB_PASSWORD || 'cm1990131',
  database: process.env.DB_NAME || 'college_recommendation'
};
```

启动时设置环境变量：

```bash
set DB_PASSWORD=your_secure_password&& node server.js
```
