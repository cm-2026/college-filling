# 登录保护安全配置说明

## 概述

本系统已实现完整的登录保护功能，采用 **JWT (JSON Web Token) + 前端双重验证** 的安全机制。

## 核心功能

### 1. 后端认证机制
- **JWT Token生成**：登录成功后返回加密的Token（有效期7天）
- **Token验证中间件**：所有API接口都需要验证Token
- **自动Token刷新**：Token过期后需要重新登录

### 2. 前端认证管理
- **统一认证管理模块**：`js/auth.js` 提供完整的Token管理功能
- **LocalStorage存储**：Token和用户信息安全存储在浏览器本地
- **自动Token注入**：API请求自动添加Authorization头

### 3. 安全特性
- **密码加密**：使用bcryptjs加密存储用户密码
- **Token过期机制**：7天后自动失效，需重新登录
- **会话管理**：登出功能清除本地存储的Token

## 安装和配置

### 1. 安装依赖

确保已安装以下依赖：

```bash
cd backend
npm install
```

需要安装的依赖包：
- `jsonwebtoken` - JWT Token生成和验证
- `bcryptjs` - 密码加密
- `mysql2` - MySQL数据库连接
- `cors` - 跨域支持

### 2. 配置环境变量

创建 `.env` 文件（已在项目中创建）：

```env
# JWT密钥（生产环境请务必修改为随机字符串）
JWT_SECRET=your-secret-key-change-in-production-please-use-a-random-string

# MySQL数据库密码
DB_PASSWORD=cm1990131
```

**⚠️ 重要安全提示：**
- 生产环境必须修改 `JWT_SECRET` 为强随机字符串
- 不要将 `.env` 文件提交到版本控制系统
- 定期更换密钥以保证安全性

### 3. 生成安全的JWT密钥

在项目根目录运行以下命令生成随机密钥：

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

然后将生成的密钥替换到 `.env` 文件中的 `JWT_SECRET`。

## 使用说明

### 用户登录流程

1. 用户在登录页面输入账号和密码
2. 前端调用 `/api/auth/login` 接口
3. 后端验证成功后生成JWT Token
4. Token和用户信息存储到浏览器LocalStorage
5. 自动跳转到首页或后台管理页面

### API请求流程

1. 前端发起API请求时，自动从LocalStorage读取Token
2. 在请求头添加 `Authorization: Bearer <token>`
3. 后端中间件验证Token有效性
4. 验证通过后执行业务逻辑

### 用户登出

点击登出按钮后：
1. 清除LocalStorage中的Token和用户信息
2. 返回登录页面

## API接口清单

### 公开接口（无需认证）
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录（返回Token）

### 需要认证的接口
所有其他API接口都需要在请求头中携带Token：
```
Authorization: Bearer <token>
```

认证失败的HTTP状态码：
- `401` - 未提供Token或Token无效
- `403` - Token已过期

## 安全最佳实践

### 1. Token安全
- ✅ Token存储在LocalStorage（生产环境可考虑使用HttpOnly Cookie）
- ✅ Token有效期设置为7天（可根据需求调整）
- ✅ Token包含用户角色信息，便于权限控制

### 2. 密码安全
- ✅ 密码使用bcryptjs加密（强度10）
- ✅ 密码长度限制6-20位
- ✅ 密码必须包含字母和数字

### 3. 环境变量安全
- ✅ `.env` 文件已添加到 `.gitignore`
- ✅ 不要在代码中硬编码敏感信息
- ✅ 生产环境使用强随机密钥

### 4. HTTPS部署
- 建议在生产环境使用HTTPS协议
- 防止Token在传输过程中被窃取

### 5. 定期维护
- 定期更换JWT密钥
- 监控异常登录行为
- 及时清理过期的Token

## 故障排查

### 问题1: 登录后Token无效

**解决方案：**
1. 检查 `.env` 文件中的 `JWT_SECRET` 是否正确
2. 清除浏览器LocalStorage重新登录
3. 检查后端日志，确认Token生成和验证是否正常

### 问题2: API请求返回401

**解决方案：**
1. 检查是否已登录
2. 查看LocalStorage中是否有Token
3. 确认Token是否过期（7天后）
4. 刷新页面重新获取Token

### 问题3: Token存储失败

**解决方案：**
1. 检查浏览器是否支持LocalStorage
2. 清除浏览器隐私设置
3. 检查浏览器控制台是否有错误信息
