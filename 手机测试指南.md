# 📱 手机测试指南

## 前置准备

### 1. 确保手机和电脑在同一局域网
- 手机和电脑连接同一个WiFi
- 或者手机开热点,电脑连接手机热点

### 2. 获取电脑局域网IP地址

**方法一：查看后端启动日志**
```bash
# 启动后端服务
cd e:\xm\backend
node server.js
```

启动后会显示类似信息:
```
🚀 后端服务器启动成功！
📡 本机访问: http://localhost:3000
🌐 局域网访问: http://192.168.1.100:3000/index-mysql.html
```

**方法二：手动查看IP**
- Windows: 打开CMD,输入 `ipconfig`,查看"IPv4 地址"
- 例如: `192.168.1.100`

## 常见问题

### 1. 手机无法访问

**检查防火墙**
- Windows防火墙可能阻止3000端口
- 解决方法: 允许Node.js通过防火墙

**临时关闭防火墙(测试用)**
```bash
# Windows CMD(管理员)
netsh advfirewall set allprofiles state off
```

**添加防火墙规则**
```bash
# Windows CMD(管理员)
netsh advfirewall firewall add rule name="高考志愿系统" dir=in action=allow protocol=tcp localport=3000
```

### 2. 端口被占用

查看3000端口占用:
```bash
netstat -ano | findstr :3000
```

结束占用进程:
```bash
taskkill /PID [进程ID] /F
```

### 3. 数据库连接失败

确保MySQL服务已启动:
```bash
# Windows
net start MySQL
```

### 4. 页面加载慢

- 检查WiFi信号强度
- 确保后端服务运行正常
- 查看浏览器控制台是否有错误

## 快速启动脚本

为了方便测试,可以创建启动脚本:

**start-mobile-test.bat**
```batch
@echo off
echo ========================================
echo 启动高考志愿系统 - 手机测试模式
echo ========================================
echo.

cd /d e:\xm\backend

echo 正在启动后端服务...
echo.
echo 启动后,请在手机浏览器访问显示的局域网地址
echo.

node server.js

pause
```

## 注意事项

1. **安全性**: 仅在可信网络环境中测试,生产环境需要配置HTTPS
2. **数据安全**: 测试数据库中的敏感信息,注意保护
3. **并发限制**: 默认连接池限制10个连接,多设备测试时注意
4. **缓存清理**: 测试前清理浏览器缓存,确保获取最新代码
