# 管理员系统使用说明

## 快速启动

### 1. 启动后端服务
```bash
cd e:\xm\backend
node server.js
```

看到以下提示表示启动成功:
```
✅ MySQL数据库连接成功！
🚀 后端服务器启动成功！
📡 本机访问: http://localhost:3000
```

### 2. 访问管理页面

**方式一：直接访问**
```
http://localhost:3000/admin.html
```

**方式二：从首页进入**
1. 访问 http://localhost:3000/index-mysql.html
2. 点击"管理员"按钮

## 功能说明

### 📊 统计面板
- **总用户数**: 系统中所有注册用户
- **今日新增**: 今天注册的用户数
- **活跃用户**: 最近7天登录过的用户
- **被禁用用户**: status=0的用户数量

### 👥 用户管理

#### 查看用户列表
- 表格显示：ID、用户名、手机号、状态、注册时间、最后登录
- 默认按注册时间倒序排列
- 每页显示20条记录

#### 搜索和筛选
- **搜索**: 输入用户名或手机号，实时搜索
- **状态筛选**: 全部/已启用/已禁用
- **重置**: 点击"重置筛选"清空所有条件

#### 用户操作

**启用/禁用用户**
- 点击"禁用"按钮 → 用户status设为0，无法登录
- 点击"启用"按钮 → 用户status设为1，恢复正常
- 操作前需确认

**重置密码**
- 点击"重置密码"按钮
- 弹窗中输入新密码（至少6位）
- 密码会被bcrypt加密存储
- 操作成功后用户可用新密码登录

**删除用户**
- 点击"删除"按钮
- 确认后永久删除该用户
- ⚠️ 此操作不可恢复，请谨慎

## 数据库结构

### users 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| phone | VARCHAR(11) | 手机号，唯一 |
| username | VARCHAR(32) | 用户名，唯一 |
| password | VARCHAR(255) | 密码（bcrypt加密） |
| status | TINYINT | 1=启用，0=禁用 |
| created_at | DATETIME | 注册时间 |
| last_login | DATETIME | 最后登录时间 |

## API接口

### GET /api/admin/users
获取用户列表

**参数**:
- `search`: 搜索关键词（可选）
- `status`: 状态筛选（可选，0或1）
- `page`: 页码（默认1）
- `pageSize`: 每页条数（默认20）

**返回**:
```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "total": 100,
    "page": 1,
    "pageSize": 20,
    "totalPages": 5
  }
}
```

### PUT /api/admin/users/:id/status
更新用户状态

**参数**:
```json
{
  "status": 1  // 1=启用, 0=禁用
}
```

### POST /api/admin/users/:id/reset-password
重置密码

**参数**:
```json
{
  "newPassword": "新密码"
}
```

### DELETE /api/admin/users/:id
删除用户

### GET /api/admin/stats
获取统计数据

**返回**:
```json
{
  "success": true,
  "data": {
    "total": 100,
    "todayNew": 5,
    "active": 50,
    "disabled": 2
  }
}
```

## 故障排查

### 问题：页面显示"服务器错误"

**检查步骤**:

1. 确认后端服务已启动
   ```bash
   # 查看是否有Node进程在运行
   tasklist | findstr node
   ```

2. 测试API连接
   ```bash
   cd e:\xm\backend
   node test-admin-final.js
   ```

3. 检查数据库连接
   - 确认MySQL服务已启动
   - 检查数据库配置（server.js中的dbConfig）

4. 查看浏览器控制台
   - F12打开开发者工具
   - 查看Console和Network标签页
   - 检查是否有API请求错误

### 问题：用户列表为空

**可能原因**:
- users表中确实没有数据
- SQL查询错误

**解决方法**:
```bash
# 检查users表数据
cd e:\xm\backend
node check-users-table.js
```
